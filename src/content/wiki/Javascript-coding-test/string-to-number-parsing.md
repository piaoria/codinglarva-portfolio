## 개요

JavaScript에서 문자열을 숫자로 변환하는 것은 매우 자주 사용되는 작업입니다. 이 문서에서는 다양한 파싱 방법과 각각의 특징을 살펴보겠습니다.

## 파싱 방법 비교

| 함수/방법                 | 변환 대상        | 예시                               | 특징                                          |
| ------------------------- | ---------------- | ---------------------------------- | --------------------------------------------- |
| `parseInt(str, 10)`       | 정수             | `"123abc"` → `123`                 | 앞부분 숫자만 추출, 정수 전용                 |
| `parseFloat(str)`         | 실수             | `"3.14abc"` → `3.14`               | 소수점 포함 가능                              |
| `Number(str)`             | 숫자 전체        | `"123"` → `123`, `"3.14"` → `3.14` | 전체가 숫자일 때만 OK, 정밀함                 |
| `+str` (단항 플러스)      | 숫자 전체        | `+"42"` → `42`                     | 가장 짧고 빠름, 직관성 떨어짐                 |
| `~~str`                   | 정수             | `~~"42"` → `42`                    | 비트 연산자 이용, 빠르지만 실수/NaN 처리 안됨 |
| `Math.floor(Number(str))` | 정수로 강제 내림 | `"3.99"` → `3`                     | 실수를 강제로 정수로 바꿔야 할 때 사용        |

## 각 방법 상세 설명

### 1. parseInt(str, radix)

```javascript
parseInt("123", 10); // 123
parseInt("123abc", 10); // 123
parseInt("abc123", 10); // NaN
```

- 두 번째 매개변수 `radix`는 진수를 지정 (기본값은 10)
- 문자열의 앞부분에서 숫자만 추출
- 정수만 변환 가능

### 2. parseFloat(str)

```javascript
parseFloat("3.14"); // 3.14
parseFloat("3.14abc"); // 3.14
parseFloat("abc3.14"); // NaN
```

- 소수점을 포함한 실수 변환 가능
- 문자열의 앞부분에서 숫자만 추출

### 3. Number(str)

```javascript
Number("123"); // 123
Number("3.14"); // 3.14
Number("123abc"); // NaN
```

- 문자열 전체가 숫자여야만 변환
- 가장 엄격한 검사 수행
- 실수, 정수 모두 변환 가능

### 4. 단항 플러스 연산자 (+)

```javascript
+"123"; // 123
+"3.14"; // 3.14
+"123abc"; // NaN
```

- 가장 간단하고 빠른 방법
- Number()와 동일한 결과
- 코드 가독성은 떨어질 수 있음

### 5. 비트 NOT 연산자 (~~)

```javascript
~~"42"; // 42
~~"3.14"; // 3
~~"abc"; // 0
```

- 비트 연산을 이용한 빠른 변환
- 실수는 소수점 이하 버림
- NaN은 0으로 변환

### 6. Math.floor(Number(str))

```javascript
Math.floor(Number("3.99")); // 3
Math.floor(Number("42")); // 42
Math.floor(Number("abc")); // NaN
```

- 실수를 강제로 정수로 변환
- 소수점 이하 버림

## 사용 시 주의사항

1. **NaN 처리**

   - 모든 방법에서 유효하지 않은 숫자는 `NaN`을 반환
   - `isNaN()` 함수로 체크 필요

2. **진수 변환**

   - `parseInt()` 사용 시 항상 radix 지정 권장
   - `parseInt("08")` → `0` (8진수로 해석)
   - `parseInt("08", 10)` → `8` (10진수로 해석)

3. **성능**
   - 단항 플러스와 비트 NOT 연산자가 가장 빠름
   - 하지만 가독성과 정확성이 더 중요한 경우가 많음

## 정리

1. **정수 변환**: `parseInt(str, 10)`
2. **실수 변환**: `parseFloat(str)`
3. **엄격한 변환**: `Number(str)`
4. **간단한 변환**: `+str`
5. **빠른 정수 변환**: `~~str`
6. **강제 내림**: `Math.floor(Number(str))`

각 방법은 상황에 따라 적절히 선택하여 사용해야 합니다. 일반적으로는 `parseInt()`와 `parseFloat()`이 가장 안전하고 명확한 방법입니다.
